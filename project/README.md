## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `notebooks/` – экспериментальные ноутбуки:
  - `eda.ipynb` - разведочный анализ данных.
- `src/` – основной код проекта:
  - `config.py` - менеджер конфигурации;
  - `preprocessing.py` - предобработка;
  - `train.py` - скрипт обучения, оценки и сохранения финальной модели;
  - `service/` - FastAPI-приложение (`app.py`, `schemas.py`, `logger.py`).
- `data/` - демонстрационные данные:
  - `bank-full.csv` - открытый датасет.
- `configs/` - конфигурационные файлы:
  - `config.yaml` - пути сервиса.
- `models/` – сохранённые артефакты (`pipeline_v1.pkl`).
- `artifacts/` – сохранённые графики, отчёты, результаты экспериментов.

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.10` 
- `pip` и `venv` 
- Docker 

### 3.2. Установка окружения

```bash
# Перейти в папку проекта
cd project

# Создать виртуальное окружение (опционально, но рекомендуется)
python -m venv .venv

# Активировать окружение:
# Windows:
.venv\Scripts\activate
# Linux / macOS:
source .venv/bin/activate

# Установить зависимости
pip install --upgrade pip
pip install -r requirements.txt
```

## 4. Как запустить проект

### 4.1. Запуск обучения модели

```bash
cd project
source .venv/bin/activate      # при необходимости
python -m src.train            # или другая команда
```

### 4.2. Запуск сервиса (API/веб-интерфейс)

```bash
cd project
source .venv/bin/activate      # при необходимости
python -m src.service          # пример: FastAPI/Flask сервис
```

Или, если используется Docker:

```bash
cd project
docker build -t aie-project .
docker run -p 8000:8000 aie-project
```

Сервис поднимается на порту 8000. Доступны эндпоинты:
- `GET /` – корневой эндпоинт с информацией о сервисе.
- `GET /health` – проверка работоспособности (возвращает {"status": "ok"} если модель загружена).
- `POST /predict` – предсказание конверсии клиента.

## 5. Данные

Для обучения и валидации используется открытый датасет UCI Bank Marketing Dataset (Moro et al., 2014). Данные полностью обезличены, не содержат PII и распространяются по лицензии CC BY 4.0.
Формат: CSV с разделителем ;, 45 211 строк × 17 признаков. В репозитории расположена небольшая часть датасета в папке `/projects/data/bank_sample`. Полный датасет доступен по ссылке: https://www.kaggle.com/datasets/adityamhaske/bank-marketing-dataset?resource=download&select=bank-full.csv. После скачивания распакуйте архив bank-full.zip и поместите файл bank-full.csv в папку data/.
Особенности: Сильный дисбаланс классов, отсутствуют пропуски. В проекте дисбаланс компенсируется через class_weight='balanced' и sample_weight.
